# Contributing

## Working Rule

This repository is part of the Septagon `platformkit` ecosystem. Changes must respect the repository charter and keep the split boundary explicit.

## Expectations

1. Do not pull legacy monorepo history into this repository.
2. Use conventional commits.
3. Keep new code within the ownership boundary defined in `REPO_CHARTER.md`.
4. Update `ROADMAP.md` or `docs/MIGRATION_BOUNDARY.md` when the boundary changes.

## Pull Requests

Every pull request should explain:
- why the change belongs in `platformkit-community`
- whether it changes the migration boundary
- whether follow-up extraction work is required elsewhere
